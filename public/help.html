Research Tab

AI-powered intelligence that finds what your audience is reading, asking, and sharing right now.

🔭 How to run research

  1. Go to Scheduler → Intelligence tab (top left)
  2. Click ⟳ Run Research — the button turns gray and shows a progress bar
  3. Wait 30–60 seconds. The AI searches for trending topics relevant to integrity, leadership, and Agile
  4. Review the findings cards. Each card shows a topic, insight, and suggested angle
  5. Click ✦ Use This on any finding to send it to the Adapter
Pro tip: Run research once a week — not every session. Each run uses Claude API tokens (~$0.10). The findings stay saved until you clear cache.
Do not click Run Research multiple times in a row — each click fires a full API call.

Content Adapter

Takes one idea and writes optimized versions for every platform in one click.

✦ How to adapt content

  1. Click ✦ Adapter tab in the top nav
  2. Paste your idea or article text into the input box — or click Use This from a research finding
  3. Select your Content Type (thought leadership, story, tip, book promotion, etc.)
  4. Click Generate — the AI writes platform-specific versions for all selected platforms
  5. Review each version. Edit inline if needed (LinkedIn allows 3000 chars, Bluesky hard limit is 300)
  6. Click ✓ Approve on each version you want to keep
Pro tip: Generate 4–8 posts in one sitting. Each costs less than $0.08. Space them 2–3 days apart when scheduling.

Approve & Queue

Your approved posts live here. Send them to Media Studio or export for Buffer.

📋 Working with the queue

  1. Click the Queue tab — shows all approved posts with platform and status
  2. Review posts. Delete any you don't want
  3. Click 🎬 Send to Media Studio to send all video posts (TikTok, YouTube Shorts, Instagram) with auto scene generation
  4. The button dims to ⏳ Generating scenes… and stays disabled until complete — do not click again
  5. For text posts: use the CSV export buttons (Buffer, Pictory, Image Prompts, Creative Brief)
Queue persists in Postgres — your posts survive app refreshes and Railway redeploys. Safe to close and reopen.

Export options

📊 Buffer CSV

Downloads a CSV with all approved posts. Upload to Buffer.com → Bulk Upload for scheduled posting on Facebook, Instagram, TikTok, X.

🎬 Pictory Scripts

Post & Schedule

Two ways to publish: direct posting for LinkedIn and Bluesky, Buffer CSV for everything else.

🚀 Direct posting (LinkedIn & Bluesky)

  1. Go to Settings → Connect tab in the Scheduler
  2. For LinkedIn: click Connect LinkedIn → authorize with your Wells Fargo / personal LinkedIn account
  3. For Bluesky: enter your handle (gopushrestha.bsky.social) and App Password (not your main password)
  4. Once connected, go back to the Adapter, approve a post, and click Publish Now
  5. The post goes live immediately — no Buffer needed
Bluesky has a hard 300-character limit. Always check the character counter before publishing.

📂 Buffer CSV (Facebook, Instagram, TikTok, X)

  1. In the Queue tab, click 📊 Buffer CSV
  2. A CSV file downloads to your computer
  3. Go to buffer.com → select your channel → Settings → Bulk Upload
  4. Upload the CSV — Buffer parses the dates and content automatically
  5. Review the scheduled posts in Buffer and confirm
Best posting times: LinkedIn 8am Monday, Instagram 11am Tuesday, TikTok 7pm any day. Set your defaults in Scheduler Settings.

Media Studio Queue

Posts arrive here from Scheduler. Generate videos and images, then download or publish.

🎬 Working with the Studio queue

  1. Open Media Studio at strategic-honesty-media-studio.pages.dev (use Firefox for now)
  2. Click Queue tab — you'll see items sent from Scheduler with their scene data
  3. Find a video item (TikTok, YouTube Shorts) — it shows the scene count and hook text
  4. Click → Generate Video — this calls the Creatomate API to render the video
  5. Poll status — takes 60–120 seconds. When done, a download link appears
  6. For LinkedIn image items, click → Create Image to generate a quote card
The queue is backed by Postgres. Items stay after refresh, browser close, or Railway redeploy.

Video Generation

Creatomate renders your 7-scene videos at 1080×1920 (portrait) for TikTok and YouTube Shorts.

🎥 How a video is built

Each video post goes through this pipeline automatically:

  1. Scene Parser — Claude generates 7 scenes: Hook, Temptation, Origin, Decision, Math, Truth, CTA
  2. Pexels Images — Each scene fetches a relevant stock image from Pexels based on the stockQuery
  3. Creatomate Render — Assembles image + caption text + overlay into an mp4 (1080×1920, 24fps)
  4. Status Poll — Check /api/video/status/:renderId every 20s until "status":"succeeded"
  5. Download URL — Backblaze B2 CDN link, valid for 7 days
Render time: 60–120 seconds for a 50-second video. Don't click Generate twice.

Troubleshooting

Common issues and quick fixes.

🐛 Known issues & fixes

Chrome shows old build / blank screen

Chrome is caching the old CRA build (content.02d4c26d.js).

Fix: Open DevTools (F12) → right-click the reload button → Empty Cache and Hard Reload. Or use Firefox — the Vite build works there now.

Video generation returns "Queue item not found"

The item ID doesn't exist in Postgres — likely a stale ID from before the Postgres migration.

Fix: Get a fresh ID from /api/queue, then retry with the real ID.

Video render fails with "element has no valid type"

A scene has an unsupported motion style or malformed animation object.

Fix: Already patched in v2 — pan_right/pan_left no longer emit animation objects. Redeploy if on an older version.

Send to Media Studio button stays active / double-fires

Old bug — let isSendingToStudio reset on every render.

Fix: Already patched — now uses useState and button is disabled while sending.

Bluesky post fails silently

Post is over 300 characters. The AT Protocol rejects it with no visible error.

Fix: Check the character counter in the Adapter before approving. Must be 300 or under — no exceptions.

Railway server shows stale data after redeploy

File-based queue (queue.json) was wiped on redeploy. Now using Postgres.

Fix: Already migrated. Queue persists in Postgres across all redeploys.
